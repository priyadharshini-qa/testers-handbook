## 🗄️ SQL for Testers

This section covers the SQL commands most useful in a QA context, including:

- `SELECT` queries to fetch and verify data
- Filtering with `WHERE`, `AND`, `OR`, and `LIKE`
- Aggregations with `COUNT`, `SUM`, `AVG`, `MIN`, `MAX`
- Joining tables to trace data relationships
- Identifying nulls, duplicates, and anomalies

📁 Browse: [`SQL-for-testers/`](./SQL-for-testers)

## 🌿 Git Commands for Testers

This section covers:

- Setting up Git (`git config`, `git init`)
- Cloning and syncing repositories (`git clone`, `git pull`, `git fetch`)
- Checking status and history (`git status`, `git log`, `git diff`)
- Working with branches (`git checkout`, `git branch`)
- Stashing and undoing changes (`git stash`, `git reset`)

📁 Browse: [`Gitcommands-for-testers/`](./Gitcommands-for-testers)

## 🚀 Getting Started

No installation needed — this is a reference guide, not a code project.

1. **Clone the repository** to keep it handy locally:
   ```bash
   git clone https://github.com/priyadharshini-qa/testers-handbook.git
   ```

2. **Navigate to the topic** you need:
   ```bash
   cd testers-handbook/SQL-for-testers
   # or
   cd testers-handbook/Gitcommands-for-testers
   ```

3. **Open any file** in your preferred editor or read it directly on GitHub.
